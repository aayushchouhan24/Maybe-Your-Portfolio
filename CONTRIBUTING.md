# Contributing Guidelines

Thank you for your interest in contributing! Please follow these rules:

- **Proposals:** For features/major changes, open an issue labeled `proposal-required` before coding.
- **Branching:** Use descriptive branch names (e.g., `feature/animation-sync`).
- **Commits:** Write clear, conventional commit messages.
- **Pull Requests:** Use the PR template. Link related issues.
- **Reviews:** Address all review comments before merging.
- **Code Style:** Follow project structure and modularity.
- **Tests:** Add/maintain tests for new features or bugfixes.
- **Docs:** Update documentation as needed.

See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for community standards.
